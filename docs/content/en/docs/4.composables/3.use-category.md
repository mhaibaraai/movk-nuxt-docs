## Overview

Manages the documentation category system, making it easy to group and navigate documentation.

### Configuring categories

Create `composables/useCategory.ts` in your project to override the default configuration:

```ts
export function useCategory() {
  return {
    categories: {
      components: [
        {
          id: 'content',
          title: 'Content Components',
          icon: 'i-lucide-component'
        },
        {
          id: 'layout',
          title: 'Layout Components',
          icon: 'i-lucide-layout-dashboard'
        }
      ],
      composables: [
        {
          id: 'navigation',
          title: 'Navigation',
          icon: 'i-lucide-compass'
        },
        {
          id: 'ui',
          title: 'UI Utilities',
          icon: 'i-lucide-sparkles'
        }
      ]
    }
  }
}
```

### Using categories in frontmatter

```md
---
title: ComponentPropsLinks
category: content
---
```

## API

### `useCategory()`{lang="ts-type"}

Returns the category configuration.

### Return value

::field-group

  :::field{name="categories" type="Record<string, Category[]>"}
  The category configuration object, keyed by documentation directory slug.
  :::collapsible
    ::field-group
      ::field{name="id" type="string" required}
      The category ID, corresponding to the `category` field in the frontmatter.
      ::

      ::field{name="title" type="string" required}
      The category display title.
      ::

      ::field{name="icon" type="string" required}
      The category icon class name.
      ::
    ::
  :::
  :::

::
