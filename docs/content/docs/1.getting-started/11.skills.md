## 什么是 Agent Skills？

Movk Nuxt Docs 会自动发现 `skills/` 目录中的技能，并遵循 [Cloudflare Agent Skills Discovery RFC](https://github.com/cloudflare/agents/blob/main/docs/skills-discovery.md) 规范，将其部署在 `/.well-known/skills/` 端点下，只需要一条命令即可在 Claude Code、Cursor 等 AI 助手中安装您的技能。

Agent Skills 是一种轻量级开放格式，用于为 AI 助手提供专属知识和工作流。每个 Skill 是一个 `SKILL.md` 文件，描述 AI 助手能用你的产品完成什么任务，以及如何完成——并可附带任意数量的参考文件。

::note{to="https://docs.mhaibaraai.cn/.well-known/skills/index.json"}
查看本文档网站发布的 Skills。
::

## 创建 Skill

::steps
### 创建 skills 目录

在项目根目录下创建 `skills/` 目录，并在其中创建一个 skill 目录，该子目录中包含一个 `SKILL.md` 文件：

```bash
my-docs/
└── skills/
    └── my-skill/
        └── SKILL.md
```

### 编写 SKILL.md

请遵循 [agentskills.io 规范](https://agentskills.io/specification)，frontmatter 中 `description` 字段为必填项，如果省略 `name` 字段，则默认使用目录名作为 Skill 名称。

```md [skills/my-skill/SKILL.md]
---
name: my-skill
description: |
  描述此 Skill 的用途。
  列举用户说什么话时应调用此 Skill。
---

# My Skill

## 工作流程

1. 第一步
2. 第二步
```

### 部署

部署后，Movk Nuxt Docs 自动将 Skills 发布到 `/.well-known/skills/`。

### 使用 Skills

用户通过以下命令将你发布的 Skills 添加到 AI 助手：

```bash [Terminal]
npx skills add https://your-docs-site.com
```

CLI 会检测已安装的代理（Claude Code、Cursor、Windsurf 等），并将该技能安装到所有这些代理上。
::

## 命名规范

Skill 名称（目录名和 `name` 字段）必须遵循[代理技能命名规范](https://agentskills.io/specification#name-field) ：

- 1-64 字符
- 仅使用小写字母、数字和连字符（ az 、 0-9 、 - ）
- 不能以连字符开头或结尾
- 不能包含连续的连字符（ -- ）
- frontmatter 中的 `name` 字段必须与父目录名完全一致

::note
未通过验证的技能将被跳过，请检查构建输出中的警告。
::

## 目录结构

Skill 目录可以包含任意数量的辅助文件，AI 助手在调用 Skill 时会按需获取：

```bash
skills/
└── my-skill/
    ├── SKILL.md
    ├── references/
    │   ├── style-guide.md      # 写作风格指南
    │   └── checklist.md        # 检查清单
    └── assets/
        └── template.md         # 页面模板
```

所有文件会自动列在 `index.json` 目录中，并在 `/.well-known/skills/{skill-name}/` 的各自路径下提供服务。

::tip
保持主 `SKILL.md` 文件在 500 行以下。将详细的参考材料移到 `references/` 中的单独文件中——代理会按需加载这些文件，因此较小的文件意味着更少的上下文使用。
::

## 发布多个 Skills

一个文档站可以发布多个独立的 Skills：

```bash
skills/
├── create-docs/    # 创建文档网站
├── review-docs/    # 审查文档质量
└── code-review/    # 代码审查
```

所有 Skills 都列在 `index.json` 中，AI 助手根据用户意图自动选择。

## 与 llms.txt 的比较

`Agent Skills 和 `llms.txt` 都帮助 AI 工具使用你的文档，但它们的目的不同：

| | llms.txt | Agent Skills |
|---|---|---|
| **目的** | 所有文档页面的目录 | 带有可操作指令的能力总结 |
| **内容** | 页面标题、描述和链接 | 分步工作流程、代码示例、约束条件 |
| **加载时机** | 发现时间 | 按需，当技能被激活时 |
| **格式** | 带链接的纯文本 | 带 YAML frontmatter 的 Markdown |
| **最佳用途** | 帮助代理找到信息 | 教会代理如何使用你的产品 |

::tip
两者一起使用：`llms.txt` 告诉代理在哪里找到信息，而 Skills 告诉代理他们能完成什么以及如何完成。
::
