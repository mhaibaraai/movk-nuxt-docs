## 什么是 Skills？

Skills 是面向 AI Agent 的能力包。每个 skill 通常包含：

- `SKILL.md`：skill 定义与工作流
- `references/`：参考文档、模板和规范
- `assets/`：示例文件或素材

Movk Nuxt Docs 会自动扫描 skills 并发布到标准发现端点，供 Cursor、Claude Code 等工具读取。

## 发现端点

默认暴露以下端点：

- `/.well-known/skills/index.json`
- `/.well-known/skills/{skill-name}/SKILL.md`
- `/.well-known/skills/{skill-name}/{file-path}`

官方站点示例：

- <https://docs.mhaibaraai.cn/.well-known/skills/index.json>
- <https://docs.mhaibaraai.cn/.well-known/skills/create-docs/SKILL.md>

## 目录结构

在文档站点中，建议使用以下结构：

```text
skills/
├── create-docs/
│   ├── SKILL.md
│   └── references/
│       ├── mdc-components.md
│       ├── configuration.md
│       └── writing-guide.md
└── review-docs/
    ├── SKILL.md
    └── references/
```

## SKILL.md 规范

每个 `SKILL.md` 必须包含 YAML front-matter：

```md [SKILL.md]
---
name: create-docs
description: 使用 Movk Nuxt Docs 创建和编写文档站点。
---
```

校验规则：

- `name` 与目录名一致
- `name` 使用 kebab-case，仅允许 `a-z`、`0-9`、`-`
- 不允许连续 `--`
- 长度不超过 64 字符
- 必须提供 `description`

## 快速使用

### 在 AI 工具中安装

```bash
npx skills add mhaibaraai/movk-nuxt-docs
```

### 直接查看已发布目录

```bash
curl https://docs.mhaibaraai.cn/.well-known/skills/index.json
```

## 与 LLMs.txt 的区别

- **Skills**：描述「可执行能力与工作流」，适合 Agent 决策与工具编排。
- **LLMs.txt**：描述「文档内容索引与知识上下文」，适合模型检索与阅读。

二者可同时使用：先通过 Skills 选择能力，再通过 LLMs.txt 获取详细知识。
