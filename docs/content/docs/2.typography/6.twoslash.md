## 基础用法

::tip
Twoslash 已内置在此 Layer 中，无需额外安装。在代码块的语言标识后添加 `twoslash` 即可启用，构建时自动注入 TypeScript 类型信息。悬停标识符可查看类型，使用 `// ^?` 注释可在对应位置内联展示类型。
::

### `hello`

```ts twoslash
console.log('Hello, world!')
```

## 项目上下文

### `nitro`

```ts twoslash [server/api/hello.ts]
export default defineEventHandler(() => {
  return { status: 'ok' }
  //         ^?
})

// @ts-expect-error should not exist in this context
const _config = defineNuxtConfig({})
```

::note
`// ^?` 注释中的 `^` 指向上方对应列，类型信息将以浮框形式展示在该位置。
::

### `node`

```ts twoslash [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ['@nuxt/content'],
  devtools: { enabled: true }
})

// @ts-expect-error should not exist in this context
const _handler = defineEventHandler(() => 'foo')
```

### `app`

使用 `[filename]` 语法切换 TypeScript 上下文，适用于 `nuxt.config.ts`、`app/*.vue`、`server/api/*.ts` 等不同文件类型：

```ts twoslash [app/app.vue]
const count = ref(0)
const route = useRoute()

definePageMeta({
  title: 'Home'
})
```

### `shared`

```ts twoslash [shared/utils.ts]
// @ts-expect-error should not exist in this context
const count = ref(0)
```

## 配置

Layer 已内置默认配置，开发环境下 Twoslash 默认关闭以提升热更新速度。如需调整：

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@movk/nuxt-docs'],

  twoslash: {
    enableInDev: true,  // 默认 false，开发时按需开启
    throws: true,       // 默认 false，类型错误时中断构建（推荐在 CI 中开启）
  },
})
```

| 选项 | 默认值 | 说明 |
|------|--------|------|
| `enableInDev` | `false`{lang="ts-type"} | 开发环境是否运行 Twoslash（关闭可加速热更新） |
| `throws` | `false`{lang="ts-type"} | 类型检查失败时是否中断构建 |

::tip{to="https://github.com/antfu/nuxt-content-twoslash/blob/main/src/module.ts"}
查看完整配置选项。
::
