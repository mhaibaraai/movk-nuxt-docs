## 启用方式

在代码块语言标签后追加 `twoslash`，构建期会调用 TypeScript 编译器为其生成富类型 HTML。未标注的代码块保持原有静态高亮，零额外开销。

::tip{to="https://github.com/antfu/nuxt-content-twoslash"}
该能力由 `nuxt-content-twoslash` 提供，类型分析在构建期完成，运行时零成本。
::

::code-preview
---
class: "[&>div]:*:my-0 [&>div]:*:w-full"
---
```ts twoslash
const count = ref(0)
//    ^?
```

#code
````mdc
```ts twoslash
const count = ref(0)
//    ^?
```
````
::

## 类型悬浮

将鼠标悬浮在任意标识符上，可查看 TypeScript 推断出的类型签名，与 IDE 中 hover 一致。

::code-preview
---
class: "[&>div]:*:my-0 [&>div]:*:w-full"
---
```ts twoslash
const count = ref(0)
const doubled = computed(() => count.value * 2)
```

#code
````mdc
```ts twoslash
const count = ref(0)
const doubled = computed(() => count.value * 2)
```
````
::

## 查询快照

使用 `//    ^?` 在代码块中固定显示某个位置的类型快照，适合在教学场景中定向展示推导链。

::code-preview
---
class: "[&>div]:*:my-0 [&>div]:*:w-full"
---
```ts twoslash
type Status = 'idle' | 'loading' | 'success' | 'error'

function unwrap<T>(value: T | null): T {
  if (value === null) throw new Error('null')
  return value
}

const result = unwrap<Status>('loading')
//    ^?
```

#code
````mdc
```ts twoslash
type Status = 'idle' | 'loading' | 'success' | 'error'

function unwrap<T>(value: T | null): T {
  if (value === null) throw new Error('null')
  return value
}

const result = unwrap<Status>('loading')
//    ^?
```
````
::

## 编译错误高亮

类型不匹配会被 TypeScript 直接捕获，错误以红色注释行渲染在代码块下方。使用 `// @errors:` 显式声明期望捕获的错误码，提示读者代码故意包含错误，避免被当成示例 bug。

::code-preview
---
class: "[&>div]:*:my-0 [&>div]:*:w-full"
---
```ts twoslash
// @errors: 2322
const message: number = 'hello'
```

#code
````mdc
```ts twoslash
// @errors: 2322
const message: number = 'hello'
```
````
::

## 与 Nuxt 类型集成

Twoslash 在构建期会读取 Nuxt 自动生成的 `.nuxt` 类型，因此 `useFetch`、`definePageMeta`、`useState` 等运行时 API 与本主题暴露的 composable 都能被直接识别——演示代码无需写显式 `import`。

::code-preview
---
class: "[&>div]:*:my-0 [&>div]:*:w-full"
---
```ts twoslash
const { primary, mode, fonts } = useTheme()

primary.value
//      ^?

mode.value
//   ^?

fonts
// ^?
```

#code
````mdc
```ts twoslash
const { primary, mode, fonts } = useTheme()

primary.value
//      ^?
```
````
::

::tip
该行为由 `nuxt.config.ts` 中 `twoslash.includeNuxtTypes` 控制，默认 `true`。
::

## 注意事项

- 仅 `ts` 与 `vue` 语言标识支持 `twoslash`，其他语言会被忽略。
- 首次构建较慢（TypeScript 加载 + 类型分析），二次走 `.nuxt/content-twoslash` 缓存通常 < 5s。
- 单独跳过某个代码块的类型分析时，去掉 `twoslash` 标签即可。
