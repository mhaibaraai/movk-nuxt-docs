## 用法

`ComponentExample` 用于在文档中嵌入可交互的组件示例。它会自动加载组件,并展示实时预览和源代码。

### 基本用法

在您的 Markdown 文档中使用 `:component-example` 指令：

::code-preview
:component-example{name="AccordionExample"}

#code
```mdc
:component-example{name="AccordionExample"}
```
::

### 高级配置

```md [md]
::component-example
---
name: AccordionExample
highlights: [10, 15, 20]
collapse: true
---
::
```

## 扩展组件 `ComponentExampleExtras`

`ComponentExample` 在示例容器内会自动渲染一个 `<ComponentExampleExtras />` 组件作为「扩展槽位」。Layer 内提供了一个空的默认实现（不渲染任何 DOM），消费方在自己仓库的 `app/components/content/ComponentExampleExtras.vue` 放同名文件即可整体替换，用于叠加 `ThemeVisualizer`、Playground 按钮等附加 UI，无需 fork 整个 `ComponentExample`。

### Props

`ComponentExample` 会把内部已经计算好的派生值透传给 `ComponentExampleExtras`，避免覆盖方重复计算：

| Prop | 类型 | 说明 |
| --- | --- | --- |
| `name` | `string` | MDC `:name="..."` 的原始值，可能是 kebab 或 camel 形式 |
| `camelName` | `string` | camelCase 化后的组件名，可直接作为 theme 查询 key |
| `pascalName` | `string` | PascalCase 名，来自 `useFetchComponentExample`，可用于组件元数据 |
| `effectiveProps` | `Record<string, any>` | 当前生效的合并 props（`componentProps + optionsValues`） |
| `options` | `Array<{ name, label, ... }> \| undefined` | `ComponentExample` 接收到的 options 配置原样透传 |
| `wrapperContainer` | `HTMLElement \| null` | 最外层 wrapper，用于 popover / 高亮框定位 |
| `componentContainer` | `HTMLElement \| null` | 示例组件实际渲染的容器，用于 `querySelector('[data-slot]')` 等 DOM 扫描 |

### 示例：叠加 ThemeVisualizer

参考 [Nuxt UI 仓库的 `ComponentThemeVisualizer.vue`](https://github.com/nuxt/ui/blob/v4/docs/app/components/content/ComponentThemeVisualizer.vue) 实现一个 `LazyComponentThemeVisualizer`，然后在覆盖文件中接入：

```vue [docs/app/components/content/ComponentExampleExtras.vue]
<script setup lang="ts">
defineProps<{
  name: string
  camelName: string
  pascalName: string
  effectiveProps: Record<string, any>
  options?: Array<{ name: string, label: string, [k: string]: any }>
  wrapperContainer: HTMLElement | null
  componentContainer: HTMLElement | null
}>()
</script>

<template>
  <LazyComponentThemeVisualizer
    :slug="camelName"
    :container="componentContainer"
    :position-container="wrapperContainer"
  />
</template>
```

### 注意事项

- 默认 stub 不渲染任何 DOM，未覆盖时对存量页面零影响。
- Layer 已在外层包过 `<ClientOnly>`，覆盖文件无需重复包裹。
- 不要在覆盖文件里再渲染 `ComponentExample`，会形成无限循环。
