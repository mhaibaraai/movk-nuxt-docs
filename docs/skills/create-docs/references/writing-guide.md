# 写作指南

有效编写文档内容的指南。

## 中文写作规范

### 标点符号

使用全角中文标点：

| 场景 | 正确 | 错误 |
|------|------|------|
| 逗号 | 内容，说明 | 内容,说明 |
| 句号 | 完成。 | 完成. |
| 冒号 | 配置：| 配置: |
| 分号 | 第一；第二 | 第一;第二 |
| 问号 | 如何使用？ | 如何使用? |
| 感叹号 | 注意！ | 注意! |
| 顿号 | 主题、布局、组件 | 主题,布局,组件 |
| 引号 | 「配置」或 "配置" | '配置' |

### 中英文混排

中文与英文、数字之间加空格：

```markdown
✅ 使用 Nuxt 4 构建文档站点
❌ 使用Nuxt 4构建文档站点

✅ 安装 @movk/nuxt-docs 包
❌ 安装@movk/nuxt-docs包

✅ 端口默认为 3000
❌ 端口默认为3000
```

数字和单位之间不加空格：

```markdown
✅ 宽度为 16px
❌ 宽度为 16 px
```

### 专有名词

保持正确大小写：

| 正确 | 错误 |
|------|------|
| Nuxt | nuxt / NUXT |
| TypeScript | typescript / Typescript |
| Vue.js | vue.js / VueJS |
| GitHub | github / Github |
| Tailwind CSS | tailwindcss / TailwindCSS |
| Markdown | markdown |
| MDC | mdc |

## 基于动作的内容

在 **H2/H3 标题** 中使用动作动词使页面易读且任务导向。

| 静态标题 | 基于动作的标题 |
|----------------|---------------------|
| 配置 | 配置你的应用 |
| 数据库设置 | 连接数据库 |
| 路由保护 | 保护你的路由 |
| 会话管理 | 处理用户会话 |
| 错误处理 | 优雅地处理错误 |
| 部署 | 部署你的应用 |

### 要使用的动作动词

**主要：** 添加、配置、创建、设置、启用、连接、处理、自定义、部署、使用

**次要：** 构建、实现、集成、安装、定义、编写、运行、测试、调试、更新

## 页面标题

不同的部分使用不同的标题风格：

| 部分 | 风格 | 示例 |
|---------|-------|----------|
| 开始使用 | 名词 | 介绍、安装、快速开始 |
| 指南 | 主题 | 身份验证、配置、部署 |
| API | 函数/组件名称 | useSession、fetchData、Button |
| 食谱 | 动作短语 | 添加深色模式、部署到 Vercel |

动作风格存在于 **内容标题** 中，而不是页面标题或文件名中。

## 语气和声调

### 要做的事

- 使用 **主动语态** 和 **现在时**
- 用 **第二人称** 撰写（"你可以"、"你的项目"）
- 要 **直接** 和 **可操作**
- 尽可能用 **动作动词** 开始句子

### 示例

| 被动/冗长 | 主动/直接 |
|---------------|---------------|
| 配置可以通过... | 通过... 配置你的应用 |
| 建议你... | 我们建议... |
| 应该在...中创建文件 | 在...中创建文件 |
| 身份验证由...处理 | 用...处理身份验证 |

## 代码示例

### 文件路径标签

始终为代码块添加文件路径标签：

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  // ...
})
```

### 多包管理器示例

使用 `::code-group` 并显示 **首先检测到的包管理器**：

```md
::code-group
```bash [pnpm]
pnpm add package-name
```

```bash [npm]
npm install package-name
```

```bash [yarn]
yarn add package-name
```

```bash [bun]
bun add package-name
```
::
```

### 最佳实践

- 包含 **有效、可复制粘贴** 的示例
- 显示 **完整** 代码，而不是片段
- 对复杂逻辑添加 **注释**
- 使用 **现实的** 变量名称
