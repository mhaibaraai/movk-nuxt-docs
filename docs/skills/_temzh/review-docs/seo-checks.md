# SEO 审查指南

Docus 文档网站的全面 SEO 最佳实践。

## 页面标题

### 长度指南
- **最优：** 50-60 字符
- **最大：** 70 字符（搜索引擎超过此限制会截断）
- **最小：** 30 字符（太短看起来不完整）

### 最佳实践
- [ ] 包含主要关键字
- [ ] 全页面唯一
- [ ] 与 H1 匹配或明显相关
- [ ] 描述性和具体（不是"文档"或"指南"）

### 示例

| 差 | 更好 | 最好 |
|------|--------|-------|
| 安装 | 安装 | 在你的 Nuxt 项目中安装 Docus |
| 配置 | 配置 | 配置 Docus 主题设置 |
| API | API 参考 | API 参考：可组合项和助手 |

## 元描述

### 长度指南
- **最优：** 120-160 字符
- **最大：** 160 字符（在搜索结果中被截断）
- **最小：** 70 字符（太短是个错失机会）

### 最佳实践
- [ ] 自然地包含目标关键字
- [ ] 引人注目和基于操作
- [ ] 准确总结页面内容
- [ ] 每页唯一（无重复）

### 示例

✅ 好的（145 字符）："了解如何在你的 Nuxt 项目中安装和配置 Docus。使用搜索、深色模式和 AI 集成设置你的文档网站。"

❌ 太短（42 字符）："Docus 主题的文档设置指南"

❌ 太长（185 字符）："本综合指南将指导你完成在你的 Nuxt 应用中安装、配置和自定义 Docus 文档主题的完整过程——包含所有功能。"

## 标题结构

### H1 要求
- [ ] **每页单一 H1**（对 SEO 至关重要）
- [ ] 匹配或关联到 `<title>` 标签
- [ ] 包含主要关键字
- [ ] 描述性和具体

### 标题层级
- [ ] 无跳过的级别（H1 → H2 → H3，永远不是 H1 → H3）
- [ ] 逻辑流和嵌套
- [ ] 页面上的每个标题都是唯一的

### 描述性标题

避免不传达意义的通用标题：

| 通用（避免） | 描述性（使用） |
|-----------------|-------------------|
| 概览 | 身份验证概览 |
| 开始使用 | 使用 OAuth 开始 |
| 详细 | 配置详细 |
| 高级 | 高级路由模式 |
| 选项 | 配置选项 |

### 关键字优化

H2 和 H3 标题应该：
- 自然地包含相关关键字
- 回答用户问题（适当时以 FAQ 风格）
- 可扫描（用户经常跳过标题）

**示例 FAQ 风格标题：**
- "我如何轮换 API 密钥？"
- "OAuth 和 JWT 之间有什么区别？"
- "我何时应该使用服务器端呈现？"

## URL 结构

### 最佳实践
- [ ] 小写、用连字符分隔（kebab-case）
- [ ] 描述性和稳定（发布后不要更改）
- [ ] 遵循编号目录模式（`1.getting-started/`、`2.guide/`）
- [ ] 匹配内容层级
- [ ] 避免特殊字符和下划线

### 示例

✅ 好的：
- `/en/getting-started/installation`
- `/en/guide/authentication`
- `/en/api/composables`

❌ 避免：
- `/en/getting_started/installation`（下划线）
- `/en/GetStarted/Installation`（不是小写）
- `/en/p/123`（不描述性）
- `/en/docs-page`（太通用）

## 内部链接

### 策略
- 链接到相关文档页面
- 使用描述性锚点文本（不是"点击这里"或"这里"）
- 包括"后续步骤"或"相关"部分
- 将重要链接添加到 frontmatter `links` 数组

### 锚文本最佳实践

| 差 | 更好 |
|------|--------|
| 点击这里 | 了解身份验证 |
| 阅读更多 | 配置 OAuth 提供商 |
| 见此页 | 查看 API 参考 |
| 文档 | 部署文档 |

### Frontmatter 中的链接

对于重要的相关链接，使用 frontmatter `links` 数组：

```yaml
---
links:
  - label: "API 参考"
    icon: "i-lucide-book"
    to: "/api/reference"
  - label: "GitHub 仓库"
    icon: "i-simple-icons-github"
    to: "https://github.com/..."
    target: "_blank"
---
```

## 内容长度

### 按页面类型的指南

| 页面类型 | 最小 | 最优 | 注释 |
|-----------|---------|---------|-------|
| 登陆（index.md） | 300 字 | 500-800 | Hero + 功能概览 |
| 指南页 | 400 字 | 600-1200 | 详细说明 |
| API 参考 | 200 字 | 400-800 | 简洁的技术参考 |
| 开始使用 | 300 字 | 500-1000 | 清晰的入职 |

### 部分长度
- **最小：** 100 字（少于此的是"薄内容"）
- **最优：** 200-400 字每部分（标题之间）
- **最大：** 1000 字在诀断小部分前

**红旗：**
- 不足 100 字的页面（太薄，SEO 价值低）
- 超过 1000 字没有小标题的部分（可扫描性差）

## 图像优化

### Alt 文本
- [ ] 描述性和具体（不是"图像"或"屏幕截图"）
- [ ] 自然地包含相关关键字
- [ ] 为无障碍用户描述图像内容
- [ ] 50-125 字符最优

### 颜色模式图像

始终提供浅色和深色变体：

```markdown
:u-color-mode-image{
  alt="显示用户分析的仪表板"
  light="/images/dashboard-light.png"
  dark="/images/dashboard-dark.png"
  class="rounded-lg"
  width="859"
  height="400"
}
```

### 图像文件名
- 使用描述性 kebab-case 名称
- 包含上下文：`dashboard-analytics.png` 而不是 `screenshot-1.png`

## 常见 SEO 问题

### 重复元数据
❌ 多个页面具有相同的标题或描述
✅ 每个页面都有唯一的、描述性的元数据

### 缺少元数据
❌ Frontmatter 中没有 `seo.title` 或 `seo.description`
✅ 所有页面都定义了 SEO 元数据

### 关键字填充
❌ "Docus Docus 文档 Docus 主题 Docus 指南"
✅ 自然语言，关键字在上下文中集成

### 断开的内部链接
❌ 当实际路径是 `/en/getting-started/` 时链接到 `/getting-started/`
✅ 链接与实际文件结构和语言前缀匹配

## 验证清单

为每个页面执行此检查列表：

- [ ] 标题：50-60 字符、包含关键字、唯一
- [ ] 描述：120-160 字符、引人注目、准确
- [ ] 单一 H1，匹配标题
- [ ] 标题层级是逻辑的（无跳过的级别）
- [ ] H2/H3 标题是描述性的，不是通用的
- [ ] URL 是稳定的、描述性的、小写连字符
- [ ] 内部链接使用描述性锚点文本
- [ ] 内容对实质性页面为 300+ 字
- [ ] 部分是 200-400 字标题之间
- [ ] 图像有描述性的 alt 文本
- [ ] 颜色模式图像有浅色/深色变体
