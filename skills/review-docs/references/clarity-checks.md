# 清晰度审查指南

评估 Movk Nuxt Docs 文档内容的清晰度和可读性。

## 中文写作规范

### 标点符号

使用全角中文标点，这是最常见的错误类别：

| 场景 | 正确 | 错误 |
|------|------|------|
| 逗号 | 安装，配置 | 安装,配置 |
| 句号 | 配置完成。 | 配置完成. |
| 冒号 | 说明：| 说明: |
| 问号 | 如何使用？ | 如何使用? |
| 感叹号 | 注意！ | 注意! |
| 顿号 | 组件、主题、布局 | 组件,主题,布局 |

**括号**内如果是中文内容，用全角括号（）；如果是代码或英文，用半角 ()。

### 中英文间距

中文与英文/数字之间必须有空格：

✅ `使用 Nuxt 4 构建文档`
❌ `使用Nuxt 4构建文档`

✅ `安装 @movk/nuxt-docs 依赖`
❌ `安装@movk/nuxt-docs依赖`

✅ `端口默认为 3000`
❌ `端口默认为3000`

**例外：** 数字与单位之间不加空格
✅ `宽度为 16px`，不写成 `宽度为 16 px`

### 专有名词大小写

| 正确 | 常见错误 |
|------|----------|
| Nuxt | nuxt、NUXT |
| TypeScript | typescript、Typescript |
| Vue.js | vue.js、VueJS |
| GitHub | github、Github |
| Tailwind CSS | tailwindcss、TailwindCSS |
| Markdown | markdown |
| MDC | mdc |
| pnpm | PNPM、Pnpm |

## 语气和风格

### 主动语态

✅ 好：「在 `nuxt.config.ts` 中添加配置」
❌ 避免：「配置可以通过 `nuxt.config.ts` 来添加」

✅ 好：「运行以下命令安装依赖」
❌ 避免：「以下命令可以用来安装依赖」

### 称呼

使用「您」而非「你」或「用户」：

✅ `您可以在 app.config.ts 中自定义主题`
❌ `你可以在 app.config.ts 中自定义主题`
❌ `用户可以在 app.config.ts 中自定义主题`

### 避免冗余

| 冗余表达 | 简洁表达 |
|----------|----------|
| 接下来，我们需要打开 `nuxt.config.ts` 文件，然后添加以下配置 | 在 `nuxt.config.ts` 中添加：|
| 通过以下步骤可以完成配置 | 配置步骤如下：|
| 请注意，您需要确保... | 确保... |

## 动作导向标题

### 规则

**指南/教程页面的 H2/H3 标题**使用动词开头：

| 静态标题（避免）| 动作标题（推荐）|
|----------------|----------------|
| 依赖安装 | 安装依赖 |
| 项目配置 | 配置项目 |
| 主题自定义 | 自定义主题 |
| 组件覆盖 | 覆盖默认组件 |
| AI 聊天集成 | 集成 AI 聊天 |
| 错误处理 | 处理错误 |
| 部署 | 部署到生产环境 |

**常用动词：**
- 主要：安装、配置、创建、启用、连接、自定义、部署、使用、覆盖
- 次要：构建、集成、定义、添加、更新、调试、优化

### 例外情况（保持名词）

- **文件名**：始终使用 kebab-case
- **快速开始区**：「简介」、「安装」、「快速开始」、「配置」
- **API 参考**：使用函数/组件名称（`useTheme`、`AiChatPanel`、`aiChat`）
- **概念页**：「架构」、「核心概念」、「最佳实践」

## 段落结构

### 长度建议

- 每段 2-5 句话
- 相邻标题之间 100-400 个中文字符
- 单句不超过 50 个中文字符（超出则拆分）

### 单一主题原则

每个段落聚焦一个主题，避免一段内混杂多个概念：

❌ 冗杂：
```
配置 AI 聊天需要在 nuxt.config.ts 中设置 aiChat 选项，同时还要配置 AI Gateway
API Key 环境变量，并且 app.config.ts 中有多个 UI 自定义选项可以调整浮动输入框
的显示位置、快捷键绑定和 FAQ 问题列表等功能。
```

✅ 清晰（拆分为多段）：
```
在 nuxt.config.ts 中配置 aiChat 选项启用 AI 聊天功能。

运行前需要设置 AI_GATEWAY_API_KEY 或者 NUXT_AI_GATEWAY_API_KEY 环境变量。

通过 app.config.ts 中的 aiChat 配置项自定义界面：包括浮动输入框、
快捷键和 FAQ 问题列表。
```

## 代码示例质量

### 质量检查清单

- [ ] 完整可复制粘贴（不是代码片段）
- [ ] 有文件名标注（如 ` ```ts [nuxt.config.ts] `）
- [ ] 有语言标识符
- [ ] 变量名有意义（不用 `foo`/`bar`）
- [ ] 命令行示例使用 `[Terminal]` 标注

### 好的代码示例

````md
```ts [nuxt.config.ts]
export default defineNuxtConfig({
  extends: ['@movk/nuxt-docs'],

  aiChat: {
    model: 'openai/gpt-5-nano',
    models: ['openai/gpt-5-nano', 'deepseek/deepseek-v3.2'],
  },
})
```
````

### 常见代码示例问题

❌ **无文件名：**
````md
```ts
export default defineNuxtConfig({})
```
````

❌ **不完整（只有片段）：**
````md
```ts [nuxt.config.ts]
// ... 添加以下内容
aiChat: { ... }
```
````

❌ **使用占位符而非真实值：**
````md
```ts [nuxt.config.ts]
export default defineNuxtConfig({
  aiChat: {
    model: 'YOUR_MODEL_HERE',
  },
})
```
````

## 常见清晰度问题

### 技术术语未定义

❌ 「使用 SSR 可以提升性能」
✅ 「使用服务端渲染（SSR）可以提升初次加载性能」

（首次出现时提供全称，后续可简写）

### 指代不明

❌ 「这会影响性能」（"这"指什么？）
✅ 「该配置会影响构建性能」

### 步骤顺序不清

❌ 把结果放在操作前
✅ 操作 → 结果：「运行 `pnpm dev` 启动开发服务器，浏览器将自动打开 http://localhost:3000」
