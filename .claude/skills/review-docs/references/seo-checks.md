# SEO 审查指南

Movk Nuxt Docs 文档站点的 SEO 最佳实践，针对中文内容进行了适配。

## 标题优化（title）

### 长度建议（中文适配）

中文字符的信息密度约为英文的 2 倍，建议：

| 状态 | 字符数 | 说明 |
|------|--------|------|
| 过短 | <8 个字 | 信息量不足，如「配置」 |
| 最优 | 10-20 个字 | 搜索引擎最佳显示范围 |
| 过长 | >25 个字 | 搜索结果会被截断 |

### 标题质量

✅ 好的标题：
- 「配置 AI 聊天功能」
- 「安装 Movk Nuxt Docs 文档主题」
- 「自定义主题颜色和字体」

❌ 需要改进：
- 「配置」（太短，不描述性）
- 「文档」（过于通用）
- 「Movk Nuxt Docs 完整安装配置自定义主题颜色字体指南」（太长）

### title vs seo.title

`title` 用于页面显示，`seo.title` 用于搜索引擎优化（可以不同）：

```yaml
---
title: AI 聊天           # 导航/页面标题（简短）
seo:
  title: 为文档站点配置 AI 聊天助手    # 搜索引擎标题（更描述性）
---
```

## 描述优化（description）

### 长度建议（中文适配）

| 状态 | 字符数 | 说明 |
|------|--------|------|
| 过短 | <30 个字 | 搜索结果摘要不够吸引 |
| 最优 | 50-80 个字 | 搜索引擎摘要最佳范围 |
| 过长 | >100 个字 | 搜索结果会被截断 |

### 描述质量

✅ 好的描述：
```
为您的文档站点添加内置 AI 聊天助手，支持多模型切换、FAQ 问题列表和自定义界面。通过 AI Gateway 接入 OpenAI、DeepSeek 等主流模型。
```

❌ 需要改进：
```
AI 聊天功能介绍。  ← 太短，无信息量
```

### 描述写作要点

- 包含页面核心关键词
- 描述页面能帮用户解决什么问题
- 使用动词开头（「为...添加」、「了解如何...」、「配置...」）
- 每页描述唯一，不重复

## 标题层级（H1-H3）

### 规则

Movk Nuxt Docs 中 H1 由系统自动从 `title` 字段渲染，文档内容从 `##`（H2）开始：

- [ ] 不在文档正文使用 `#`（H1）
- [ ] 不跳级（`##` → `###`，不从 `##` 跳到 `####`）
- [ ] H2 和 H3 标题描述性强，不通用

### 描述性标题

| 通用（避免）| 描述性（推荐）|
|-------------|---------------|
| 概述 | AI 聊天功能概述 |
| 配置 | 配置模型和 API Key |
| 示例 | 自定义浮动输入框示例 |
| 选项 | aiChat 配置选项参考 |

## 内部链接

### 链接文本质量

✅ 描述性锚文本：
- 「了解更多关于 [AI 聊天配置](/docs/ai-chat)」
- 「查看 [app.config.ts 配置参考](/docs/configuration)」

❌ 无意义锚文本：
- 「点击 [这里](/docs/ai-chat) 了解更多」
- 「[查看详情](/docs/configuration)」（太通用）

### 链接覆盖

- [ ] 每个指南页面有指向相关页面的链接
- [ ] Front-Matter `links` 字段包含主要相关资源
- [ ] 章节入口页链接到本章所有子页面

## 内容长度

### 建议长度（中文）

| 页面类型 | 最少字数 | 最优字数 |
|----------|----------|----------|
| 首页（index.md） | 200 字 | 400-600 字 |
| 快速开始/安装 | 300 字 | 500-800 字 |
| 功能指南 | 400 字 | 600-1200 字 |
| API 参考 | 200 字 | 400-800 字 |

相邻 H2 标题之间：100-400 个中文字符。

**警告：**
- 少于 100 字的章节：内容太薄，SEO 价值低
- 超过 1000 字的章节无子标题：拆分为 H3 子章节

## 图片 SEO

### Alt 文本

所有图片必须有描述性 alt 文本：

✅ 好：`alt="Movk Nuxt Docs 文档站点截图，展示 AI 聊天面板"`
❌ 差：`alt="截图"`、`alt="图片"`

### 深浅模式图片

使用 `:u-color-mode-image` 提供两个版本：

```mdc
:u-color-mode-image{
  alt="AI 聊天面板截图"
  light="/images/ai-chat-light.png"
  dark="/images/ai-chat-dark.png"
  class="rounded-lg"
  width="1200"
  height="675"
}
```

### 图片文件命名

使用描述性 kebab-case 命名：
✅ `ai-chat-panel-dark.png`
❌ `screenshot-1.png`、`图片.png`

## URL 结构

Movk Nuxt Docs 自动从文件结构生成 URL，数字前缀不出现在 URL 中：

| 文件路径 | 生成 URL |
|----------|----------|
| `1.getting-started/1.index.md` | `/docs/getting-started` |
| `1.getting-started/2.installation.md` | `/docs/getting-started/installation` |
| `2.guide/ai-chat.md` | `/docs/guide/ai-chat` |

**URL 规则（自动保证）：**
- 全小写
- 连字符分隔（kebab-case）
- 无特殊字符
- 描述性路径

## 常见 SEO 问题

| 问题 | 影响 | 修复建议 |
|------|------|----------|
| 多页面 title 相同 | 搜索引擎降权 | 每页设置唯一 title |
| description 过短（<30 字）| 搜索摘要不完整 | 补充描述，说明页面价值 |
| 缺少 seo.title/seo.description | 搜索优化欠缺 | 为重要页面添加 SEO 专用字段 |
| 标题不描述性（「配置」、「指南」）| 搜索点击率低 | 添加具体内容说明 |
| 链接锚文本「点击这里」 | SEO 信号弱 | 改为描述性文本 |
